# Skill 升级工作流

## 来源
- [x] 协作发现（未失败验证）

## 触发条件

- 你下载了别人的 skill 或官方 skill，想让它真正"属于你"
- 一个 skill 被你用过几次，有了自己的判断
- 一条规则你违反了，吃了亏，需要升级权重

---

## 升级路径

```
阶段 0：下载导入
└── 标注来源 = "AI归纳" 或 "他人真实代价"

阶段 1：初次使用
└── 用完后追加一条使用记录（见格式）

阶段 2：验证
└── 用过 3 次以上且有效 → 来源升级为"协作发现（未失败验证）"

阶段 3：内化
└── 违反过并吃亏，或严格遵守避开了坑 → 升级为"真实代价"
    → 规则加粗，补充损失描述
```

---

## 使用记录格式

在每个 skill 文件末尾追加此区块：

```markdown
## 使用记录
- YYYY-MM-DD 场景：xxx，结果：有效/无效/需修改，备注：xxx
```

---

## 升级触发条件

| 触发事件 | 操作 |
|---------|------|
| 用了 3 次，每次有效 | 来源 → 协作发现（未失败验证） |
| 遇到边界情况，规则需要调整 | 修改规则，记录为什么改 |
| 违反了，出了问题 | 来源 → 真���代价，规则加粗，写清楚损失 |
| 30 天内从未用到 | 考虑删除或归档 |

---

## 执行步骤

1. 打开目标 skill 文件
2. 检查 `## 来源` 字段，确认当前阶段
3. 在 `## 使用记录` 区块追加本次记录
4. 判断是否满足升级条件，满足则修改来源字段
5. 如升级为"真实代价"，加粗相关规则并补充损失描述

---

## 注意事项

- 别人的真实代价 ≠ 你的真实代价，初始权重仍按"AI归纳"处理
- 升级不可降级，一旦标注为"真实代价"，不要轻易删改规则
- 使用记录不需要长，一句话即可，关键是留下时间和结果
